Best practices

Realtime updates

There is need to display actual information to client on buy, sell or expiration of contract.

It is possible to use API-methods like portfolio, profit_table, statement, balance, but all they are "poll-model", i.e. slow and impose additional load on Binary.com, and as the cummulative effect might lead to lags or errors. Those calls are designed to be invoked at application initialization stage.

The best way to be notified about contract execution or balance update is subscribe to transaction. All the relevant information can be extracted from the transaction notification on money deposite/withdraw or contract buy/sell execution.

For example to implement real-time portfolio updates using transaction, you can:

  1. If transaction returns a new contract with action ‘buy’, add it to portfolio
  2. If transaction returns an existing contract with action ‘sell’, remove it from portfolio

As the transaction notification will be delived as soon as an event happen, the user interface will appear more reactive, comparing to "poll-methods" invocation.

Please note, that to prevent missing transaction notification, at application inititialization it should subscribe to transactions first, then actualize current state (statement, portfolio balance), and only then allow user to preform relevant actions.
