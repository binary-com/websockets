FAQ

What is the easiest way to get started with Binary.com API?

View code samples. You can use these code snippets in your apps to open a connection to the Binary.com WebSocket API service.

What is the easiest way to build my own Binary.com front-end application?

Simply copy our open-source code, and adapt it to meet your needs. Follow these steps to get started:

  1. Open a GitHub account.
  2. Download the GitHub Desktop application.
  3. Fork one of Binary.com's front-end repositories, such as Binary Webtrader (view app) or any of the other open-source front-end repositories listed here.
  4. Make your code changes in your fork.
  5. Publish your fork using the GitHub Pages facility.
  6. For SSL and website acceleration for your app, open a free account on Cloudflare.

Will I earn commissions if users signed up to Binary.com using my app?

Yes. Please follow these steps:

  1. Sign up as an affiliate.
  2. Insert your affiliate token into the new_account_virtual call in your app.

Can someone build an app for me?

Sure! You can hire developers on sites such as UpWork. Just look for developers who are familiar with JavaScript and WebSocket technology.

What does it mean if a schema property has the "sensitive" attribute?

This means we will treat the value of this property as confidential, and will never return it in any API response. It is used for passwords and tokens.

How do I get help?

Please visit Binary.com API dev forum or email api-support@binary.com.
